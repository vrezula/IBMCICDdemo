Google Cloud Fundamentals: Core Infrastructure

### Intro 
  Colocation came first. Then virtualization. Recently, containers.
  IaaS and PaaS. Cloud networks - Regions divided into Zones.
  Data centers were first to have ISO 14001 certification
  Employees use U2F Universal Second Factor accounts. Why not DFA? Too early?
  Google Cloud Observability can monitor performance across clouds.

## Resources and Access in the Cloud
  ProjectID is a globally unique ID used by Google. Like a UUID.
  Project Number is also unique. Not Project name, though of course.
  Folders inherit policies from the project. 
  IAM - Identity and Access Management - goog acct, group, serv acct. cloudID
  Basic roles - owner, editor,viewer, billing admin.  Custom IAM roles, too
    Serv Accts can be given to a VM.     Cloud Identity manages permissions.
  Google CLI includes cli and bq - a tool for BigQuery.  Cloud Shell is Debian
  
## Virtual Machines and Networks
  Sustained-use and Committed-use discounts. discount at 25% use. Up to 57%.
  VPC features - routing tables. no external IP needed. Firewall w rules.
    VPC Peering to see others. Shared VPC exists, too.
  Application load balancer - layer 7. can do external and internal LBing.
  Layer 7 load balancers can inspect the content of the traffic.
  Peering - putting a router in the same datacenter as a google PoP. 
     point of presence. There are more than 100 PoPs. 
	Carrier Peering is done on a Service Provider's network. no SLA.
	Dedicated Interconnect is covered 99.9% by SLA.  Partner Interconnect.
    Cross-Cloud - connects entirely separate cloud provider. 10/100 Gbps
  Subnets have regional scope.
  
## Storage in the Cloud
  Cloud Storage = object storage. GUIDs are in URL format. Object versioning.
    ACL = access control lists for finer than IAM control.
    4 storage classes. Standard. Nearline Storage (1 access/month)
	  Coldline (1 access/quarter) Archive(1 access/year)  Autoclass!
	Transfer appliance - up to a petabyte of data.
  Cloud SQL = for relational databases MySQL, PostgreSQL, SQL Server. <64TB
  Spanner = SQL, horizontally scales. High IOPs. petabytes
  Firestore = NoSQL, horizontally scales document storage. indexed.
    max of 1MB per entity. 
  Bigtable = NoSQL, big data. High throughput. good for IoT. >1TB.
    max 10 MB/cell 100 MB/row
